USER INSTRUCTIONS – ePOSTER VIEWER
OVERVIEW
This application is an offline ePoster viewer designed for conferences, exhibitions, and kiosk displays.
It displays poster images in a responsive grid (Index page) and provides a clean, distraction‑free Viewer with keyboard and mouse navigation.
The system supports optional metadata (author name and poster title) provided via a CSV file.
No server, no database, and no internet connection are required.

BASIC FEATURES
• Offline use (opens directly from file system)
• Responsive grid of poster thumbnails
• Full‑screen poster viewer
• Keyboard navigation (Left / Right arrows)
• Mouse wheel navigation
• On‑screen navigation arrows
• Clean kiosk‑style layout
• Fast loading using image thumbnails
• Optional author & title metadata

ADDING POSTERS

Prepare poster images

• Supported formats: JPG / JPEG / PNG
• Recommended orientation: portrait
• Recommended naming format:
NNN. Poster Name.jpg
Examples:
001. Poster Land.jpg
002. Urban Mornings.jpg
010. Digital Pathology.jpg
The number at the beginning defines the display order.


Place images

Copy all poster images into the materijal folder.


Generate the poster list

Inside the materijal folder, run:
build_manifest.bat
This creates:
• manifest.js – list of poster files in correct order
Run this again whenever posters are added, removed, or renamed.

OPTIONAL: ADDING AUTHORS AND POSTER TITLES
Author names and poster titles can be added using a CSV file.
If no CSV is provided, the system works normally and displays poster names based on file names.

CSV FORMAT
File name: titles.csv
Location: materijal/
Encoding: UTF‑8 (without BOM recommended)
Required columns (exact names):
ID,name,title
Example:
ID,name,title
001,Author 1,Just some land during spring
002,Author 2,Urban mornings
010,Author 3,Digital pathology systems
Rules:
• ID must match the numeric prefix of the poster file
• name is the author’s name
• title is the poster title
• Commas inside fields are NOT supported

GENERATING TITLES METADATA
After creating or editing titles.csv, run:
build_titles_js.bat
This creates:
• titles.js – metadata file used by the viewer
You only need to rerun this script if:
• titles.csv is changed
• new posters are added
• IDs are modified

HOW TITLES ARE DISPLAYED
INDEX PAGE (thumbnail grid):
If titles.js exists:
• First line: Author name
• Second line: POSTER TITLE (ALL CAPS)
If titles.js does NOT exist:
• Poster name is derived from the file name

VIEWER PAGE:
• Header title: Poster title (from titles.js)
• Right side of navigation bar: Author name
• If metadata is missing, file name is used as fallback

VIEWER CONTROLS
Keyboard:
• Left Arrow – Previous poster
• Right Arrow – Next poster
Mouse:
• Scroll up – Previous poster
• Scroll down – Next poster
Navigation bar:
• TABLE OF CONTENTS – return to index
• Arrow buttons – previous / next poster

IMPORTANT NOTES
• Always regenerate manifest.js after changing posters
• Always regenerate titles.js after changing titles.csv
• Poster IDs must stay consistent between filenames and CSV
• The application is designed for long‑running kiosk use
• No server or browser permissions are required
